# Viona Kaleb - Portfolio

A code-editor-styled portfolio built with React, Vite, TypeScript, Tailwind, and Framer Motion. Modeled after the Jaxon Cruz Framer layout, rethemed to a purple/dark-navy palette.

## Stack

- **Vite + React 18 + TypeScript** for the build and runtime
- **Tailwind CSS** with a custom design system in `tailwind.config.js`
- **Framer Motion** for tab transitions, section reveals, and the active-section underline
- **Medium RSS** via `api.rss2json.com` for the blog tab (no backend needed)

## Run

```bash
npm install
npm run dev      # http://localhost:5173
npm run build    # production bundle in dist/
npm run preview  # serve the built bundle locally
```

## Project structure

```
src/
  data/
    resume.json      # all profile content, edit here to update the site
    types.ts         # TypeScript interfaces for resume.json
  lib/

  hooks/

    useLiveTime.ts        # ticking Jakarta-time string

  components/
    Sidebar.tsx      # left rail: avatar, bio, contact lines, CTAs
    TopBar.tsx       # tabs (viona.info / work.done / blog.share) + live status
    LineGutter.tsx   # the code-editor line-number rail
    IndexNav.tsx     # right rail with scroll-spy
    Section.tsx      # animated section wrapper with code-comment header
    TabHeaders.tsx   # the big display titles per tab
    StatusBar.tsx    # bottom strip with social links
    icons.tsx        # inline SVG icon set
  sections/
    HeroSection.tsx, AboutSection.tsx, EducationSection.tsx,
    StackSection.tsx, EducationSection.tsx, ExperienceSection.tsx,
    AwardsSection.tsx, TestimonialsSection.tsx,
    BlogSection.tsx, ContactSection.tsx
  App.tsx            # tab routing + layout shell
  main.tsx           # entry
  index.css          # Tailwind + base styles
```

## Where to edit

- **Content** lives in `src/data/resume.json`. Change a project title, swap stack levels, update work history - all here.
- **Colors** live in `tailwind.config.js` under `theme.extend.colors`. Tweak `accent.DEFAULT` to shift the purple.
- **Sections per tab** are wired in `src/App.tsx` inside `InfoView`, `WorkView`, `BlogView`. Reorder freely.
- **Medium username** is hard-coded at the top of `src/lib/medium.ts`.

## Known gaps

- No unit tests yet. The code skill would call for Vitest covering hooks and the Medium fetcher.
- Profile photo is a "VK" gradient tile, not a real image. Drop a real photo into `public/` and reference it in `Sidebar.tsx` and `AboutSection.tsx`.
- The `Frontend-Resume-Viona-Kaleb.pdf` download link in `Sidebar.tsx` expects a file at `public/Frontend-Resume-Viona-Kaleb.pdf`. Add it or the button 404s.
- Medium RSS goes through `api.rss2json.com` (free, has rate limits). For higher traffic, replace with your own serverless function.
- No accessibility audit run yet (axe, Lighthouse).

## Design notes

- The line-number rail in the main content is generated by `LineGutter`, which measures the scroll-height of the content column with a `ResizeObserver` and lays out enough numbers to cover it.
- Tab transitions use `AnimatePresence mode="wait"` so the old tab fully unmounts before the next mounts. This avoids duplicate-id collisions for sections reused across tabs (e.g. `stack`, `awards`).
- The active-section underline in `IndexNav` is a single Framer Motion `layoutId` element that animates between list items.
